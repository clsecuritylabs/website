Framework

The Closed Loop

A four-layer trust framework for AI security assessment and remediation, powered by the CLAP adapter protocol.

Four Layers

Attack. Verify. Remediate. Certify. The loop runs until the model is proven safe — or proven it can’t be.

01

Adapters — CLAP Protocol

How tools connect. The CLAP protocol defines standardized interfaces for any red team scanner (Garak, PyRIT, promptfoo, custom), any defense framework (NeMo Guardrails, Llama Guard, custom), and any verification judge (Gemini, Claude, Llama, local). Make your tool interoperable in under 100 lines. pip install clap-protocol

02

Verification

The trust layer. Every breach observation passes through multi-vendor LLM judge evaluation, semantic deduplication, and statistical reproduction testing. We’ve demonstrated that breach rates vary 41.9–67.1% depending on which vendor’s model judges the findings. Nothing reaches Layer 3 or Layer 4 without Layer 2 certification.

03

Orchestration

How the loop runs. Attack → verify → reproduce → remediate → deploy → re-verify → iterate. Each cycle generates structured training data that makes the next cycle smarter. Confidence-based deployment: ≥0.90 auto-deploy, 0.70–0.90 monitored, <0.70 human-in-the-loop.

04

Certification

How findings become proof. Certified Remediation Patterns (CRPs) carry full verification provenance — which judges agreed, reproduction rates across temperatures, defense performance per tier, and compliance mappings. CRP-2026-0147 means: this attack was reproduced, this defense blocks it, three vendors confirmed it.

The CLAP Protocol

Closed Loop Adapter Protocol — the open connector standard that powers Layer 1.
Like USB for AI security tools. Your tool speaks CLAP, it works with everything.

Attack Adapters

Garak, PyRIT, promptfoo, Forge Engine, HiddenLayer, custom scripts — any tool that generates adversarial prompts.

Judge Adapters

Gemini, Claude, Llama, GPT-4, local models — any LLM that can evaluate whether a response is a genuine breach.

Defense Adapters

NeMo Guardrails, Llama Guard, AWS Bedrock Guards, Sovereign Guardrail, custom classifiers — any defense layer.

Conformance Levels

CLAP defines what must be verified, not how. Any conforming implementation is CLAP-compliant.

Basic

1+ attack adapter, 1+ judge model, single assessment pass, findings report.

Standard

2+ adapters, 2+ judges from different vendors, reproduction testing, defense verification, CRPs with provenance.

Enterprise

3+ vendors for judges, cross-attacker validation, kernel enforcement, CRP Registry with full audit trail, compliance export.
